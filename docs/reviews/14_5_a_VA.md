## Phần 3 — Root Cause Tổng Hợp

Tất cả các lỗi trên có thể quy về **3 root cause kỹ thuật**:

### RC-1: Không có "Capability Guard" — LLM không biết mình không có data

Synthesizer hiện chỉ có `ES_EMPTY_WARNING` (khi ES log rỗng) và `DATA_SOURCE_WARNING` (khi Prometheus N/A). Không có guard cho "loại data này platform hoàn toàn không có access".

### RC-2: Global Health Header không bị suppress đủ mạnh

`[health_hint: 🔴]` luôn được inject vào context, khiến LLM luôn có xu hướng output header bảng overview dù câu hỏi không cần.

### RC-3: Intent Routing không cover câu hỏi incident với filter phức tạp

`_INCIDENT_COUNT_RE` và `_FIND_INCIDENTS_RE` không bắt pattern "tồn đọng quá Xh", "quá hạn SLA". Không có intent `INCIDENT_SLA_QUERY` để route sang endpoint incidents/stats với filter time.

### Fix #1 — Capability Guard: Từ chối câu hỏi vượt scope (CRITICAL, effort LOW)

**Vị trí:** `synthesizer.py` — thêm detection trước khi format context

**Ý tưởng:** Classify câu hỏi thuộc loại nào, nếu cần data platform không có → inject explicit guard thay vì để LLM hallucinate.

```python
# synthesizer.py — thêm function mới

_DOCKER_KW = frozenset({
    "container", "docker", "image", "dockerfile", "exited", "oomkilled",
    "dangling image", "docker ps", "running container",
})
_APPLOG_KW = frozenset({
    "module kế toán", "module kho", "module mua hàng", "module bán hàng",
    "erp module", "scheduled job", "batch import", "báo cáo tự động",
    "đồng bộ dữ liệu", "session timeout", "user đăng xuất đột ngột",
})
_SLOWQUERY_KW = frozenset({
    "slow query", "slow log", "query chậm", "query tốn thời gian",
    "slow_query_log",
})
_BACKUP_KW = frozenset({
    "backup", "sao lưu", "restore", "recovery", "backup thất bại",
    "backup thành công",
})
_PII_KW = frozenset({
    "pii", "thông tin nhạy cảm", "password trong log", "token trong log",
    "mask", "che thông tin",
})
_RETENTION_KW = frozenset({
    "retention", "log quá hạn", "xóa log cũ", "ilm policy",
    "log retention policy",
})

_CAPABILITY_GUARDS: list[tuple[frozenset, str]] = [
    (_DOCKER_KW,     "container_runtime"),
    (_APPLOG_KW,     "app_business_log"),

    "container_runtime": (

        "Em chỉ đọc được log text trong Elasticsearch — không thể lấy trạng thái "
        "container (Exited, OOMKilled), danh sách image, hay docker ps output. "
        "Để kiểm tra: `docker ps -a`, `docker events`, hoặc cấu hình cAdvisor → ES."
    ),
    "app_business_log": (

        "ES index hiện tại không có field này — cần cấu hình ERP ghi log theo format "
        "có field `module` hoặc `job_name` vào ES."
    ),
    "slow_query_log": (
        "Em không có access vào MySQL slow query log. "
        "Slow query log cần được export vào ES (qua Filebeat) với index riêng, "
        "sau đó cấu hình `slow_query_index` trong Admin → Datasource."
    ),
    "backup_monitoring": (
        "Em không có thông tin backup từ hệ thống monitoring. "
        "Backup status cần được ghi vào ES (qua script báo cáo kết quả) "
        "hoặc integrate với Prometheus job=backup_job."
    ),
    "pii_scanner": (
        "Phát hiện PII trong log cần DLP tooling hoặc regex scan toàn bộ log — "
        "không thể thực hiện realtime qua chat. "
        "Đề xuất: dùng Elasticseach Watcher với regex pattern, "
        "hoặc tool chuyên dụng như Nightfall, Google DLP."

    "ilm_policy": (

        "`GET /_ilm/policy` — em không có quyền gọi endpoint này. "
        "Anh/chị có thể kiểm tra trực tiếp trong Kibana → Stack Management → ILM."
    ),
}


def _build_capability_guard(question: str, context: dict) -> str:
    """Return explicit guard message if question requires data platform doesn't have."""
    q_lower = question.lower()
    for keywords, guard_type in _CAPABILITY_GUARDS:
        if any(kw in q_lower for kw in keywords):
            msg = _CAPABILITY_MESSAGES[guard_type]
            return (
                f"[CAPABILITY_GUARD: Câu hỏi này vượt ngoài scope của platform]\n"
                f"{msg}\n"
                f"BẮT BUỘC: Thông báo rõ ràng cho user rằng em không có data này. "
                f"KHÔNG được ước tính hay suy diễn con số cụ thể.\n\n"
            )
    return ""
```

```python
# Hiện tại (synthesizer.py _format_single):

# Fix: thêm condition — chỉ inject health_hint khi intent thực sự là health/overview
```

## Phần 6 — Bài Học Rút Ra

### 1. Hallucination xảy ra khi LLM "muốn hữu ích" hơn là "muốn chính xác"

LLM được train để luôn trả lời, không bao giờ nói "tôi không biết" trừ khi được instruction rõ ràng. Platform cần **explicit hard stops** — không phải soft hints.

### 2. "Không có data" phải là signal rõ ràng, không phải rỗng

Khi ES trả về 0 kết quả, LLM không biết đó là vì:
- (a) Hệ thống đang hoạt động tốt, hoặc
- (b) Platform không có index/access cho loại data này

Cần phân biệt 2 trường hợp này ở tầng query executor, không để LLM tự suy.

### 3. Format template cứng làm LLM "nhét" câu hỏi vào template sẵn

Bảng global overview là format mặc định → LLM render nó cho mọi câu hỏi. Cần format suppression per intent và per question type.

### 4. Các câu hỏi vận hành thực tế phức tạp hơn test cases

Session này cho thấy operator thực sự hỏi rất đa dạng:
- Docker runtime, backup status, slow query → cần data sources bổ sung
- Incident SLA với threshold phức tạp → cần routing riêng
- Module-level ERP breakdown → cần schema agreement với ERP team

Platform cần **documented capability boundary** — danh sách rõ những gì có thể và không thể hỏi.
